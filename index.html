    Smart Copy Machine

    Designed & built a gantry-style pen plotter utilizing stepper motors, driver chips, cameras, and computer vision all running or controlled with a Raspberry Pi 4B+ to scan and replicate hand drawn images by processing images into a series of coordinate-based motor commands.

    Hardware Overview:

    For the hardware side there are three axes to deal with. X and Y are the motion of the pen. While the Z axis is the up and down motion of the draw plate (whiteboard). The X and Y axes have one stepper motor per axis that is connected via belt to a carriage allowing it to move along linear rails. The Z axis is controlled by two stepper motors that raise and lower via a screw mechanism.

    Software Overview:

    When the user is done drawing their image, the camera captures a frame that begins the image processing script. In this script the image runs through some basic image processing and is skeletonized leaving contours that get converted into XY coordinates for the motors to follow. We used Bresenham's algorithm for smooth diagonal movements. Once the coordinates are found they are put into a list creating a path that the pen can follow and includes 'breakpoints' allowing for the pen needs to get picked up from the whiteboard if drawing disconnected lines. These coordinates are turned into motor movement by pulsing GPIO pins sending step counts and direction data to the MP6500 Driver Chips.

    Search and Rescue Robot

    The end goal of a robotics programming lab I took in the spring of 2025 was to program a search and rescue robot to autonomously navigate an unknown environment to find 'victims' (paper blocks), pick them up and carry them back to the starting location. My team and I (myself and two others) lovingly named our robot "Alfred". The core of Alfred is a Rock Pi running ROS 2 (Robot Operating System) Iron with additional MCUs, Gyroscopes, a USB camera, two motors and wheels, a two-joint arm, etc.

    Key Features:

  • Autonomous Navigation: Alfred's main feature is his ability to navigate through rooms, around desks, and through doors. This was done by creating and exporting a map of the rooms he searched in Blender then placing AprilTags around the room for localization. A fixed search path was provided to Alfred to follow but if something was in the way, or if Alfred drifted off course onboard pathfinding systems would calculate a path for him.
  • 'Victim' (Box) Detection and Retrieval: The boxes Alfred was searching for had AprilTags on them allowing Alfred to find them with his camera. Once the box position pose is found, Alfred postions himself in order to pick up the box with his arm.
  • Return to Home: After finding and picking up a box, Alfred would use his localization and path finding systems to return to his starting position, then drop off the box before starting his search for a box again.

    Custom Keyboard PCB

    As a personal project, I designed and built a fully custom mechanical keyboard PCB using KiCad. The keyboard was inspired by the "Nullbits NIBBLE" keyboard which I loved. A 65% keyboard kit they send you to solder and assemble yourself. After that keyboard eventually died due to a crack in the PCB that caused a trace to disconnect, I decided to design my own PCB from scratch with exactly all the features I wanted.
    Some of the main features of this keyboard include both USB-C and Bluetooth connectivity, three fully programmable rotary encoders, four programmable macros, and some other features all on a custom '70ish%' layout I made myself.

    Key Features:

  • USB-C and Bluetooth Connectivity: The keyboard can be used wired with a USB-C cable or wirelessly via Bluetooth at the flick of a switch. Having Bluetooth connectivity means there is also a rechargeable battery on the PCB which should provide up to 5 hours of battery life. Battery charging circuitry is from the MCU I chose, the Nice!Nano V2
  • Rotary Encoders and Macros: There are three rotary encoders on the left side keyboard. I plan on using one for main volume control while the other two can be for volume control of specific apps. Four Macros are found on the right side of the keyboard, and can be programmed to do anything via the keyboard firmware I am using, ZMK
  • Other Features: The keyboard also has RGB underglow lighting, a small programmable I2C OLED screen to display useful info. It also includes a TRRS port I can use to connect expansion boards in the future if I want to add more features (Like a dedicated numpad). The port has four conductors in it, two of which are for i2c data, while the other two for 5V and GND lines.
  • MCU and Firmware: As mentioned the MCU I am using is the Nice!Nano V2, it allows for seamless Bluetooth and wired connection. For firmware I am using, ZMK, allows for full programmability of all keys and features of the keyboard.

    "Aperture Analysis"

    This was the capstone for my Computational Physics Independent Study, I did at Syracuse University in the fall of 2024. This project explored the relationship between diffraction spike patterns in stellar images and the shape of a telescope's mirror or lens aperture. The goal was to reconstruct the aperture geometry of an optical system by analyzing diffraction spikes in images of stars taken with that system. The project involved developing a C++ program that used a custom Fast Fourier Transform (FFT) and Inverse FFT (IFFT) implementation to analyze the frequency components of the stellar images and to isolate the diffraction spike patterns in the fequency domain. The program was then able to use the diffraction spike angles to calculate and reconstruct the aperture shape.

    Key Features:

  • Custom FFT and IFFT Implementation: A custom Fast Fourier Transform (FFT) and Inverse FFT (IFFT) algorithm was implemented in C++ to analyze the frequency components of stellar images. This allowed for the isolation of diffraction spike patterns from the images.
  • Diffraction Spike Analysis: The program analyzed the angles and intensities of the diffraction spikes in the stellar images. This data was used to infer the geometry of the telescope's aperture.
  • Aperture Reconstruction/Visualization: Once the Aperture was reconstructed, the data was piped into a Python script where it was visualized using Python's Matplotlib library to create a 2D representation of the aperture shape to display to the user.

    Celestial Veil

    Over the summer of 2024, I worked to refine my ability to tackle complex programming challenges. Taking inspiration from Sebastian Lague and his solar system project in particular, I set off to recreate a physically accurate solar system in the Unity Game Engine using C#. This is part of a larger plan I have of developing and publishing a video game named Celestial Veil with this solar system being the playable environment for it.

    Physically Accurate Orbital Mechanics

  • By far the biggest challenge was programming physically accurate orbits. After some research I was able to implement a N-body Simulation in C#. This is done by applying forces to each orbital body based on Newton's law of universal gravitation. This project taught me a lot about the limits of accuracy with computers as well as how to deal with issues such as floating point precision errors.
